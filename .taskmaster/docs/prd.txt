# YakuLingo — M365 Copilot Translation App: Translation Result Display Fix

## Overview
YakuLingo is a bidirectional translation application built on M365 Copilot.
It translates Japanese to English (with multiple style options) and other
languages to Japanese (single result with explanation). Translation is
performed via M365 Copilot chat UI automated through Edge WebDriver (Playwright).

## Problem Statement
Three issues prevent the app from becoming operational:

### Issue 0: Startup Freeze at "準備中..." Screen (ACTIVE)
アプリ起動時に「準備中...」画面でフリーズし、翻訳可能状態に遷移しない。

**原因分析:**
GPTモード自動選択は既にno-op化済み（`ensure_gpt_mode()`, `ensure_gpt_mode_required()`,
`_ensure_gpt_mode_setup()` すべて即座にreturn）。

起動時のフローは以下の通り:
1. `connect()` → Edge起動 + CDP接続 + ログインチェック → 成功
2. `_wait_for_copilot_ready()` (app.py:2290) → `check_copilot_state()` をポーリング
   - タイムアウト: **3600秒（1時間）** — `RESIDENT_STARTUP_READY_TIMEOUT_SEC`
3. `check_copilot_state()` (copilot_handler.py:6347) → URLが/chatでも
   チャット入力欄セレクタが見つからないと `LOADING` を返す (line 6421-6429)
4. `wait_for_prompt_ready()` (copilot_handler.py:6804) → 入力欄の可視性チェック
   - タイムアウト: **300秒** — `RESIDENT_STARTUP_PROMPT_READY_TIMEOUT_SEC`

**フリーズの直接原因候補:**
- `_check_copilot_state()` line 6421-6429: `/chat` URLにいるが `CHAT_INPUT_SELECTOR_EXTENDED`
  が見つからず `LOADING` を返し続ける → ポーリングが最大1時間続く
- Copilotページの読み込みが遅い/完了しない
- `/landing` や `/landingv2` から `/chat` へのリダイレクトが行われない

**Copilotチャット入力欄の最新HTML (2026-03-31時点):**
```html
<span role="textbox" contenteditable="true" tabindex="0"
      aria-label="Copilot にメッセージを送信する"
      id="m365-chat-editor-target-element"
      class="fai-EditorInput__input ..."
      data-placeholder="Copilot にメッセージを送信する"
      data-lexical-editor="true">
```
→ セレクタ `#m365-chat-editor-target-element, [data-lexical-editor="true"]` はマッチするはず。
→ タイミング問題（DOM未描画）の可能性が高い。

### Issue 1: GPT Mode Selection Freeze (RESOLVED)
GPTモード自動選択は既にno-op化済み。
- `ensure_gpt_mode()` → 即座にreturn
- `ensure_gpt_mode_required()` → `True` を即座にreturn
- `_ensure_gpt_mode_setup()` → 即座に `copilot_ready=True`, `CONNECTED` に設定

### Issue 1.5: New Chat Button Selector Update
Copilot UIの「新しいチャット」ボタンの要素が変更された可能性がある。

**現在のコード上のセレクタ** (`copilot_handler.py:1199`):
```
#new-chat-button, [data-testid="newChatButton"], button[aria-label="新しいチャット"]
```

**最新のCopilot UIのボタン要素** (2026-03-31時点):
```html
<button type="button"
        data-automation-id="newChatButton"
        data-testid="newChatButton"
        aria-label="新しいチャット"
        class="fui-Button r1f29ykk ...">
```

**変更点:**
- `id="new-chat-button"` が削除されている
- `data-automation-id="newChatButton"` が新規追加
- `data-testid="newChatButton"` と `aria-label="新しいチャット"` は維持

現在のセレクタはOR条件(`data-testid` / `aria-label`)でマッチするため
即座に壊れるわけではないが、廃止された `#new-chat-button` を削除し、
新しい `data-automation-id` セレクタを追加して堅牢性を高めるべき。

### Issue 2: Translation Results Not Displayed
Translation results are not displayed in the app UI. Instead, only the
placeholder text "翻訳結果がここに表示されます" is shown. This may be a
downstream consequence of Issue 1 (translation never completes), or an
independent issue in the result capture/parse/display pipeline.

## Architecture Summary

### Data Flow
```
User Input (text or file)
  → AppState (state.py)
    → TranslationService (translation_service.py)
      → CopilotHandler (copilot_handler.py) → M365 Copilot via Edge/Playwright
        → Raw response text
      → Response parsing → TextTranslationResult with .options[]
    → AppState.text_result = result
  → UI refresh → text_panel.py renders result
```

### Key Files
- `app.py` (628 KB) — Main application logic, state management, translation orchestration
- `yakulingo/ui/state.py` — AppState dataclass
- `yakulingo/ui/components/text_panel.py` — Text translation result panel
- `yakulingo/ui/components/file_panel.py` — File translation result panel
- `yakulingo/services/translation_service.py` — Translation orchestration, result parsing
- `yakulingo/services/copilot_handler.py` (505 KB) — M365 Copilot browser automation
- `yakulingo/models/types.py` — Data types (TextTranslationResult, TranslationOption)

### UI Display Condition
The result panel in `text_panel.py:645` renders results only when:
```python
if state.text_result and state.text_result.options:
```
If either is falsy, the empty placeholder "翻訳結果がここに表示されます" is shown.

## Suspected Root Causes (to investigate)

### 0. Startup "準備中..." Freeze (ACTIVE)
- `_wait_for_copilot_ready()` が `check_copilot_state()` をポーリングし続ける
- `check_copilot_state()` がチャット入力欄を見つけられず `LOADING` を返す
- タイムアウトが3600秒（1時間）と長大
- **要調査: なぜチャット入力欄がDOMに存在しないのか、タイミングか構造変更か**

### 1. GPT Mode Selection Freeze (RESOLVED)
- 既にno-op化済み（ensure_gpt_mode, ensure_gpt_mode_required, _ensure_gpt_mode_setup）
- フリーズの原因ではない

### 2. Copilot Response Not Captured
- `copilot_handler.py:_get_response()` (~line 8272) may return empty
- Copilot error patterns may be triggered (COPILOT_ERROR_PATTERNS)
- Connection/timeout issues with Edge WebDriver

### 3. Response Parsing Failure
- `translation_service.py:_parse_style_comparison_result()` (multi-style for English)
- `translation_service.py:_parse_single_translation_result()` (single for Japanese)
- If parsing fails, `result.options` will be an empty list

### 4. UI State Not Updated
- `app.py:~10134-10136` — result assignment to `state.text_result`
- `_refresh_result_panel()` not triggered
- Client context lost during `asyncio.to_thread()`

### 5. CSS/Layout Visibility
- `.has-results` class not applied (app.py:~7461-7470)
- Result panel hidden by CSS rule (styles.css:2436-2475)

### Issue 3: Startup Performance — Overall Slowness

起動全体に時間がかかっている。既にNiceGUI遅延import、Playwright並列初期化、
Early Edge起動など最適化が入っているが、タイムアウト値やポーリング間隔に
改善の余地がある。

**起動フロー概要:**
```
1. Early Edge startup (daemon thread)     ← 並列
2. NiceGUI import (~2.6-2.8s)            ← 並列
3. Playwright pre-init (background)       ← 並列
4. Early Copilot connection (background)
5. Main page render
6. _wait_for_copilot_ready() polling      ← ボトルネック
7. wait_for_prompt_ready()                ← ボトルネック
```

**ボトルネック一覧 (最大待ち時間):**

| 順位 | 箇所 | ファイル:行 | 最大待ち | 備考 |
|------|------|------------|---------|------|
| 1 | Edge起動ポーリング | copilot_handler.py:2156 | **20秒** | 0.25s×80回 |
| 2 | チャットUI待ち | copilot_handler.py:3987 | **13秒** | 1s+2s×6回 |
| 3 | CDP接続リトライ | copilot_handler.py:2682 | **6.5秒** | 指数バックオフ |
| 4 | Context取得リトライ | copilot_handler.py:2904 | **3秒** | 0.3s×10回 |
| 5 | 準備中ポーリング | app.py:2012 | **3600秒** | Issue 0 |
| 6 | Prompt Ready待ち | copilot_handler.py:6804 | **300秒** | 0.8sポーリング |

**最適化方針:**

**A. Edge起動ポーリング高速化** (copilot_handler.py:2156)
- 現在: `EDGE_STARTUP_CHECK_INTERVAL=0.25s`, `MAX_ATTEMPTS=80` → 最大20秒
- 改善: 間隔を0.1sに、最大回数を50に → 最大5秒
- CDPポート応答があれば即座に次ステップへ

**B. チャットUI待ち短縮** (copilot_handler.py:3987-4035)
- 現在: `FIRST_STEP=1s`, `STEP=2s`, `MAX_STEPS=7` → 最大13秒
- 改善: `MAX_STEPS=5` に削減 → 最大9秒
- ログイン済みユーザーは1-2ステップで完了するため影響小

**C. CDP接続リトライ高速化** (copilot_handler.py:2660-2691)
- 現在: 初期0.5s, バックオフ1.5x → 最大6.5秒
- 改善: 初期0.2s, バックオフ1.5x, 上限1.5s → 最大4秒

**D. Context取得リトライ高速化** (copilot_handler.py:2873-2904)
- 現在: `RETRY_COUNT=10`, `RETRY_INTERVAL=0.3s` → 最大3秒
- 改善: 回数5, 間隔0.2s → 最大1秒

**E. 準備中ポーリング短縮** (app.py:2012) — Issue 0と同一
- 現在: `RESIDENT_STARTUP_READY_TIMEOUT_SEC=3600`
- 改善: 60-120秒

**F. Prompt Ready待ち短縮** (copilot_handler.py:6804)
- 現在: `RESIDENT_STARTUP_PROMPT_READY_TIMEOUT_SEC=300`
- 改善: 60秒

**既に最適化済みの箇所** (変更不要):
- NiceGUI遅延import (app.py内でのみimport, ~6秒節約)
- Settings遅延読み込み (初回クライアント接続時)
- Playwright並列初期化 (NiceGUI importと同時)
- Early Edge起動 (デーモンスレッドで並列)

4. Maintain existing functionality (glossary, bilingual output, style options)

## Key Data Types
```python
TextTranslationResult:
    source_text: str
    options: list[TranslationOption]  # Must be non-empty for display
    output_language: str              # "en" or "jp"
    detected_language: str
    error_message: Optional[str]

TranslationOption:
    text: str          # Translated text

```

## Success Criteria
- After a text translation request, the result panel shows the translated text
- For JP→EN: Three style options (standard, concise, minimal) are displayed
- For other→JP: Single translation with explanation is displayed
- Error states show meaningful error messages, not the empty placeholder
