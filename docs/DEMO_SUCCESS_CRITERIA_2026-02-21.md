# デモ成功基準（固定）

作成日: 2026-02-21  
用途: 将来構想（To-Be）に対する受け入れ基準（Go/No-Go 判定）

## 1. 品質KPI

- 基準: 翻訳サンプル **10件中8件以上**で合格（>= 80%）
- 合格判定: 用語の正確性、意味の保持、業務でそのまま使える自然さ
- 記録: `sample_id / result(ok/ng) / reason` を残す

## 2. 速度KPI

- 基準: **1リクエスト5秒以内**
- 測定単位: APIリクエスト開始から最終レスポンス受信までの秒数
- 判定: 各テストケースで `latency_sec <= 5.0`

## 3. 利用KPI

- 基準: **同僚5人 / 合計100リクエスト以上**
- 記録: 利用者ID（匿名可）ごとのリクエスト数
- 判定: `unique_users >= 5` かつ `total_requests >= 100`

## 4. 判定ルール

- 3指標すべて達成で「デモ成功」
- 1つでも未達なら「追加改善して再計測」

## 5. 追跡テンプレート（最小）

```text
[QUALITY]
samples_total=10
samples_ok=8

[LATENCY]
case_01=4.2
case_02=4.9
...

[USAGE]
unique_users=5
total_requests=100
```
